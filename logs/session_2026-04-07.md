26. **First run was wrong on Q2** because I assumed z-up. The numbers
    were absurd (some episodes showing 0.000 m camera-to-table). Re-checked
    episode 0's actual numeric values: camera y ≈ 1.07, wrist y ≈ 0.89,
    cam y barely moves while wrist y varies. **EgoDex / ARKit is y-up**, not
    z-up. The §1.1 snippet's `n = [0, 0, 1]` is also wrong (it would point
    along the depth axis in a y-up frame). Patched the script to use y as
    the up axis. Captured the bad first run as
    `logs/runs/2026-04-07_111000_calibrate_open_questions.log` so the
    z-up vs y-up sanity check is on the record.

27. **Re-ran the corrected script** → `logs/runs/2026-04-07_111200_calibrate_open_questions_v2.log`.

    **Q1 — confidence distribution (806,775 samples)**:
    - Range is genuinely [0, 1]. NOT a low/medium/high enum (9,202 distinct
      values across all samples).
    - **Bimodal**: a sharp untracked-floor mode at 0.000–0.003 (15.58% of
      samples sit at exactly 0.0032; another 2.30% at 0.0000), and a broad
      tracked mode spanning ~0.2 to 1.0.
    - Percentiles: p05=0.003, p25=0.257, p50=0.614, p75=0.816, p95=0.940.
    - Threshold sweep — fraction kept above the threshold:
      0.1 → 82.0%, 0.2 → 78.8%, 0.3 → 72.0%, 0.5 → 58.0%, 0.7 → 42.0%, 0.9 → 9.3%.
    - **Recalibration**: use **0.1** for the hard "is this joint tracked at
      all" filter (cleanly cuts the floor mode without dipping into real
      data). Reserve **0.5** for confidence-*weighted* IK (contribution
      pitch #1), not for hard rejection. Compute per-joint, not averaged
      across the hand.

    **Q2 — camera-to-table distance**:
    - With y-up: per-episode mean median 0.243 m, p10/p90 0.171 / 0.312 m,
      global frame range 0.105 – 0.727 m, std across episodes 0.068 m.
    - Only 0.4% of episodes have a per-episode mean within ±20% of 0.5 m.
      So `table_dist = 0.5` is wrong by ~2× for `basic_pick_place`.
    - Bigger insight: we don't need a constant. We have per-frame camera
      poses, so we can compute `d_t = |cam_y_world[t] - table_y|` exactly.
      Estimate `table_y` once per episode as the 5th percentile of the
      active wrist's world-y (with optional ~5 cm bias for the
      wrist-above-fingertips offset). Per-frame d makes the homography
      formula closer to the geometry that's actually in the data.
